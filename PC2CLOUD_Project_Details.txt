
PC2CLOUD — COMPLETE PROJECT OVERVIEW
====================================

PROJECT IDEA
============

PC2CLOUD is a platform that converts a normal personal computer into a private cloud storage and remote access server.

Users install a desktop client on their PC, connect it to their account, and then access files remotely through a web dashboard or another device.

Instead of storing files on expensive third-party cloud servers, files remain on the user's own computer while the platform handles:

- Authentication
- Device management
- Secure encrypted communication
- File metadata
- Remote access connectivity
- Optional relay services

The system works similarly to:
- Personal NAS
- Google Drive (private version)
- Remote file access platform

Main advantages:
- Privacy-focused
- Lower storage costs
- Uses existing hardware
- Scalable architecture
- Suitable for students, offices, hospitals, freelancers, and small businesses

=====================

- Remote access connection
- Reduce server bandwidth usage

Connection Flow:
User Device A
    ->
Signaling Server
    ->
User Device B

After handshake:
Direct encrypted connection established.

Fallback:
If direct connection fails:
Relay server mode will be used.

WHY NODE.JS OVER PYTHON
=======================

Node.js is preferred for this project because:

1. Same language everywhere
   - Frontend
   - Backend
   - Desktop app

2. Excellent realtime support
   - WebSockets
   - Socket.IO
   - Streaming

3. Electron uses Node.js internally

4. Better ecosystem for:
   - File streaming
   - WebRTC
   - Desktop integration
   - Async networking

5. Faster development speed

6. Easier debugging and maintenance

MAIN SYSTEM FLOW
================

1. User creates account
2. Desktop client logs in
3. Device registers with server
4. Server stores device details
5. Dashboard displays online device
6. User accesses files remotely
7. Files transfer through:
   - P2P connection
   OR
   - Relay server fallback


DEVICE REGISTRATION FLOW
========================

Each user gets:
- User ID
- Device ID
- Authentication token

Server stores:
- device_id
- user_id
- public_key
- last_seen
- device_status
- IP metadata

BIGGEST TECHNICAL CHALLENGES
============================

1. NAT Traversal
   - Hardest networking component
   - Solved using STUN/TURN servers

2. File Sync Conflicts
   - Same file edited in multiple places
   - Requires version tracking

3. Large File Transfers
   - Requires chunk uploads
   - Resume support
   - Hash verification

4. Secure Connectivity
   - End-to-end encryption
   - Token validation
   - Secure sessions
